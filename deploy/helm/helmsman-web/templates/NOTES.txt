Helmsman web is deployed as {{ include "helmsman-web.fullname" . }} in namespace {{ .Release.Namespace }}.

1. Open the UI:
{{- if .Values.ingress.enabled }}
   https://{{ (index .Values.ingress.hosts 0).host }}
{{- else }}
   kubectl -n {{ .Release.Namespace }} port-forward svc/{{ include "helmsman-web.fullname" . }} 8787:{{ .Values.service.port }}
   open http://localhost:8787
{{- end }}

2. Cluster access (kubectl):
{{- if .Values.kubeconfig.secretName }}
   Using mounted kubeconfig Secret '{{ .Values.kubeconfig.secretName }}'.
{{- else }}
   Using the pod ServiceAccount '{{ include "helmsman-web.serviceAccountName" . }}'.
   {{- if .Values.rbac.clusterAdmin }}
   Bound to cluster-admin (full functionality: install/scale/delete/etc.).
   {{- else if .Values.rbac.byoClusterRole }}
   Bound to your ClusterRole '{{ .Values.rbac.byoClusterRole }}'.
   {{- else }}
   rbac.create=false or read-only — mutations/installs may be denied.
   {{- end }}
{{- end }}

3. AI chat (the Helmsman copilot):
{{- if or .Values.claude.token .Values.claude.existingSecret }}
   Claude token configured. Chat should work.
{{- else }}
   No Claude token set — chat is disabled until you add one.
   Either: helm upgrade ... --set claude.token=sk-ant-oat-... (from `claude setup-token`),
   or set it from the in-app Settings screen (persisted to the claude volume).
{{- end }}

4. Auth — Helmsman can scale/delete/install across the cluster.
{{- if .Values.auth.disable }}
   WARNING: auth.disable=true — running with NO login. Anyone who can reach this
   can run cluster-admin commands. Only do this on a fully trusted/private network.
{{- else if .Values.auth.existingSecret }}
   Browser login enabled (password from Secret '{{ .Values.auth.existingSecret }}').
{{- else if .Values.auth.password }}
   Browser login enabled with the password you set.
{{- else }}
   Browser login enabled with a GENERATED password. Retrieve it:
     kubectl -n {{ .Release.Namespace }} get secret {{ include "helmsman-web.fullname" . }}-auth -o jsonpath='{.data.HELMSMAN_PASSWORD}' | base64 -d; echo
{{- end }}
   For SSO instead, set auth.disable=true and front it with oauth2-proxy / Cloudflare Access.
